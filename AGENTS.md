# Workspace Rules

- Workspace Root 是 `E:\Workspace`。
- 一个项目对应一个文件夹。
- 所有项目放在 `E:\Workspace\projects`。
- 所有产出必须进入对应项目，不把项目文件散落在 Workspace 根目录。
- 开始工作前先确定当前项目。
- 优先读取当前项目的 `AGENTS.md`、`README.md` 和 Workspace 的 `SUMMARY.md`。
- Search First：先针对目标文件、目录、关键词搜索，不无目的扫描整个 Workspace。
- 按需读取：不一次加载全部历史、知识和日志。
- 用户最新明确指令优先。
- 删除、覆盖、安装软件、修改系统配置、连接外部服务前必须确认。
- 完成任务后更新项目 `README.md` 和必要摘要。
- 用户指出错误，或产生值得复用的经验时，记录到 `memory`。
- 禁止自行扩大任务范围。
